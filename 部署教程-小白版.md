## 🚀 部署到GitHub详细步骤

### 第一步：检查GitHub CLI是否已登录

打开终端（命令行），输入：

```bash
gh auth status
```

**如果显示已登录**，会看到类似这样的信息：
```
✓ Logged in to github.com as YOUR_USERNAME
```
那就可以直接跳到第二步。

**如果显示未登录**，会看到：
```
You are not logged into any GitHub hosts. To log in, run: gh auth login
```
那就需要先登录，输入：

```bash
gh auth login
```

然后按照提示操作：
1. 选择 `GitHub.com` （按回车）
2. 选择 `HTTPS` （按回车）
3. 选择 `Yes` 来用GitHub凭证认证Git（按回车）
4. 选择 `Login with a web browser` （按回车）
5. 会显示一个8位数的代码，比如：`ABCD-1234`
6. 按回车，会自动打开浏览器
7. 在浏览器中输入那个8位数代码
8. 点击"Authorize github"按钮
9. 看到成功提示后，回到终端

### 第二步：进入部署目录

```bash
cd /home/ubuntu/xingyao666-deploy
```

### 第三步：创建GitHub仓库并推送代码

**方法A：使用GitHub CLI自动创建（推荐）**

输入以下命令：

```bash
gh repo create xingyao666 --public --source=. --remote=origin --push
```

这个命令会：
- 创建一个名为 `xingyao666` 的公开仓库
- 自动添加远程仓库地址
- 自动推送代码到GitHub

如果成功，会看到类似这样的输出：
```
✓ Created repository YOUR_USERNAME/xingyao666 on GitHub
✓ Added remote https://github.com/YOUR_USERNAME/xingyao666.git
✓ Pushed commits to https://github.com/YOUR_USERNAME/xingyao666.git
```

**方法B：手动创建（如果方法A不行）**

1. 打开浏览器，访问：https://github.com/new
2. 在"Repository name"输入：`xingyao666`
3. 选择"Public"（公开）
4. **不要**勾选"Add a README file"
5. 点击绿色的"Create repository"按钮
6. 回到终端，输入以下命令（注意替换YOUR_USERNAME为你的GitHub用户名）：

```bash
git remote add origin https://github.com/YOUR_USERNAME/xingyao666.git
git branch -M main
git push -u origin main
```

### 第四步：启用GitHub Pages

**方法A：使用GitHub CLI**

```bash
gh repo edit --enable-pages --pages-branch main
```

**方法B：在网页上手动设置**

1. 打开浏览器，访问你的仓库：`https://github.com/YOUR_USERNAME/xingyao666`
2. 点击页面上方的 **Settings**（设置）标签
3. 在左侧菜单中找到 **Pages**，点击它
4. 在"Build and deployment"部分：
   - Source选择：**Deploy from a branch**
   - Branch选择：**main**
   - 文件夹选择：**/ (root)**
5. 点击 **Save** 按钮
6. 等待1-2分钟，刷新页面
7. 会看到一个绿色的提示框，显示：
   ```
   Your site is live at https://YOUR_USERNAME.github.io/xingyao666/
   ```

### 第五步：配置自定义域名

#### 5.1 在GitHub上设置域名

1. 还是在 **Settings → Pages** 页面
2. 找到"Custom domain"部分
3. 在输入框中输入：`xingyao666.top`
4. 点击 **Save** 按钮
5. 等待DNS检查（可能需要几分钟）
6. 检查通过后，勾选 **Enforce HTTPS**（强制HTTPS）

#### 5.2 在域名服务商配置DNS

现在需要去你购买域名的网站（比如阿里云、腾讯云、GoDaddy等）配置DNS记录。

**添加以下A记录：**

| 记录类型 | 主机记录 | 记录值 | TTL |
|---------|---------|--------|-----|
| A | @ | 185.199.108.153 | 600 |
| A | @ | 185.199.109.153 | 600 |
| A | @ | 185.199.110.153 | 600 |
| A | @ | 185.199.111.153 | 600 |

**可选：添加www子域名（推荐）**

| 记录类型 | 主机记录 | 记录值 | TTL |
|---------|---------|--------|-----|
| CNAME | www | YOUR_USERNAME.github.io | 600 |

**具体操作步骤（以阿里云为例）：**

1. 登录你的域名服务商网站
2. 找到"域名管理"或"DNS管理"
3. 找到域名 `xingyao666.top`，点击"解析"或"DNS设置"
4. 点击"添加记录"
5. 按照上面的表格，添加4条A记录
6. 如果要添加www，再添加1条CNAME记录
7. 保存

**注意：DNS生效时间**
- DNS记录生效通常需要10分钟到48小时
- 国内域名通常10-30分钟就能生效
- 可以用这个网站检查DNS是否生效：https://www.whatsmydns.net/

### 第六步：等待部署完成

1. DNS配置好后，等待10-30分钟
2. 访问 `https://xingyao666.top` 看看是否能打开
3. 如果还不能访问，再等一会儿，DNS可能还没完全生效
4. 也可以访问 `https://YOUR_USERNAME.github.io/xingyao666/` 来测试GitHub Pages是否正常

## 🔍 常见问题解答

### Q1: 为什么访问域名显示404？

**可能原因：**
1. DNS还没生效，再等等
2. GitHub Pages还没部署完成，等1-2分钟
3. CNAME文件没有正确创建

**解决方法：**
- 先访问 `https://YOUR_USERNAME.github.io/xingyao666/` 看GitHub Pages是否正常
- 用 https://www.whatsmydns.net/ 检查DNS是否生效
- 确认GitHub Settings → Pages中显示"Your site is live"

### Q2: 网站可以访问，但是图片不显示？

**解决方法：**
- 确认 `starflare_logo_transparent.png` 和 `starry-hero-bg.png` 文件已经上传到GitHub
- 在浏览器中按F12，查看Console是否有错误信息
- 等待几分钟，GitHub可能还在处理文件

### Q3: 如何更新网站内容？

如果你修改了HTML文件，需要重新推送到GitHub：

```bash
cd /home/ubuntu/xingyao666-deploy
# 修改文件后
git add .
git commit -m "更新网站内容"
git push
```

等待1-2分钟，网站就会自动更新。

### Q4: 语言切换按钮不工作？

- 刷新浏览器缓存（Ctrl+F5）
- 检查浏览器Console是否有JavaScript错误
- 确认HTML文件中的JavaScript代码完整

### Q5: 域名服务商在哪里找DNS设置？

**阿里云：**
控制台 → 域名 → 域名列表 → 解析

**腾讯云：**
控制台 → 域名注册 → 我的域名 → 解析

**GoDaddy：**
My Products → Domains → DNS

**Cloudflare：**
Dashboard → 选择域名 → DNS
